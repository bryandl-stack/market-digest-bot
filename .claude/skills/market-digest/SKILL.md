# 시장 다이제스트

오늘의 시장 데이터를 크롤링하고 요약해 텔레그램 봇으로 보낸다.
아래 순서를 **정확히** 따른다. (명령 형식을 바꾸지 말 것 — cron 무인 실행의 권한 매칭에 의존)

## 1. 데이터 수집 (크롤러)

아래를 실행해 data/ 를 채운다(실패한 소스는 건너뛰고 진행):

```
python3 crawlers/crawl_telegram.py morning --countries=us,eu,jp,kr
python3 crawlers/fetch_prices.py us   # 필요한 시장 각각: us eu jp kr
python3 crawlers/fetch_prices.py eu
python3 crawlers/fetch_prices.py jp
python3 crawlers/fetch_prices.py kr
python3 crawlers/fetch_macro_calendar.py
```

## 2. 덤프 읽기

다음 명령을 실행하고 출력 전체를 읽는다:

```
python3 digest.py dump
```

## 3. 한국어 요약 작성 (5섹션)

덤프를 바탕으로 텔레그램용 한국어 요약을 작성한다. **아래 순서대로 5개 섹션**으로 만든다.
덤프의 `[📰 뉴스·레포트 원천]` 항목들은 **핵심 뉴스**와 **증권사 레포트** 두 섹션의 공통 재료다(당일 우선, 당일이 비면 전일).

> 참고: 요약 언어는 기본 한국어다. 다른 언어로 바꾸려면 이 섹션(3)의 지시문을 원하는 언어로 수정한다.

### 가독성·서식 규칙 (중요)
전송 스크립트가 `**...**` 를 텔레그램 굵은 글씨(`<b>`)로 변환하고 `& < >` 는 자동 처리한다. 따라서:
- 굵게 강조는 **`**텍스트**`** 형식만 쓴다. (HTML 태그·다른 마크다운 금지)
- **각 섹션 제목**(예: `📈 주가 등락`)은 `**...**` 로 굵게.
- 핵심 뉴스·증권사 레포트에서 **종목명, 목표주가, 투자의견 변동** 같은 핵심 키워드를 굵게.
- 섹션 사이에는 **빈 줄 1개**를 넣어 시각적으로 구분한다.
- 항목(불릿) 사이는 붙여 쓰되, 한 항목이 길면 자연스럽게 한 줄로 둔다.
- 출처 URL은 그대로 붙인다(텔레그램이 자동 링크). `**` 로 감싸지 말 것.

### 📈 주가 등락
시장별(미국/유럽/일본/한국) 상·하위 종목.

### 📰 핵심 뉴스
중요도 높은 핵심 뉴스만 추려 간결히 요약.
- **티커(종목코드)는 적지 않는다.**
- 각 항목 끝에 **출처 링크**를 단다 — 덤프의 `링크:` URL을 그대로 붙이면 텔레그램이 자동으로 클릭 링크로 만든다. (증권사 레포트로 분류한 항목은 여기서 제외)

### 🏦 증권사 레포트
원천 항목 중 **증권사가 목표주가·투자의견을 변경했거나 기업에 대해 코멘트를 남긴 것**만 골라 정리한다. 각 항목 형식(간결하게):
- **종목명** | 목표주가 $00→$00 | 투자의견 변동(예: `outperform 유지`, `neutral→outperform 상향`) | 한줄 코멘트 — 출처링크
- 목표주가/투자의견 변동은 본문에 명시된 것만. 해당 정보가 없으면 그 칸은 생략하고 핵심 코멘트만 남긴다.
- 출처 링크는 핵심 뉴스와 동일하게 `링크:` URL을 붙인다.
- 해당하는 항목이 없으면 "데이터 없음".

### 🌐 매크로
주요 지수·금리·환율·원자재(전일 대비).

### 🗓 캘린더
예정 경제지표·실적.

공통 규칙: 덤프에 **있는 사실만** 사용(과장·추측·환각 금지). 데이터 없는 섹션은 "데이터 없음". 머리말/설명 없이 **요약 본문 텍스트만** 작성.

## 4. 저장

작성한 요약을 `logs/digest_out.txt` 에 쓴다(Write). (logs/ 는 gitignore이며 항상 쓰기 가능. /tmp 는 환경에 따라 막힐 수 있어 사용하지 않는다.)

## 5. 전송

```
python3 digest.py send logs/digest_out.txt
```

`전송 완료: N자` 가 출력되면 성공. 사용자에게 도착 여부를 한 줄로 보고한다.
